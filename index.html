Autenticacao hibrida

Integre biometria digital e reconhecimento facial

O modulo de identificacao permite trabalhar em totens, tablets ou coletores moveis. A API registra hash biometrico e imagens embaralhadas, vinculando a cada entrega com nivel de confianca e blockchain opcional para auditoria.

  • SDK para sensores digitais padrao ANSI 378 e Wiegand
  • Engine de reconhecimento facial com liveness detection
  • API REST para validacao em sistemas terceiros
  • Armazenamento seguro de templates biometrico e logs hermeticos

Fluxo resumido

  1. Identificacao do colaborador via digital ou facial
  2. Verificacao de pendencias de EPI ou treinamento
  3. Registro da entrega com timestamp e hash biometrico
  4. Atualizacao imediata do estoque e emissao do comprovante

Compatibilidade com catracas, fechaduras inteligentes e sensores IOT.

Modelo relacional proposto

O esquema foi elaborado para atender operacoes multiempresa, rastreabilidade de lotes e controle de ciclo de vida de EPIs. Veja o arquivo db/schema.sql para obter o script completo de criacao das tabelas.

Nucleos principais

  • companies: matriz e filiais com hierarquia (self reference).
  • employees: colaboradores vinculados a empresas.
  • epis: estoque por lote, serial e validade de CA.
  • stock_movements: entregas, devolucoes e ajustes.
  • training_sessions & training_participants: controle de treinamentos e presencas.
  • biometric_events: armazenagem segura de metadados de autenticacao.

Regras chave

  • Constraints para garantir estoque nao negativo.
  • Triggers para atualizar saldos automaticamente.
  • Indices por CA, lote e empresa para auditoria rapida.
  • Views para dashboards de consumo e EPIs vencidos.
  • Politicas de retencao para dados biometrico conforme LGPD.

Integracoes

  • Webhook para ERPs e RH.
  • Sincronizacao com fornecedores via EDI.
  • Exportacao automatica para eSocial (S-2245).
  • APIs GraphQL e REST documentadas via OpenAPI.
